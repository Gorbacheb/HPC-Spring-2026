# Лабораторная работа 3: GPU Ray Tracer

## Обзор

В этой лабораторной реализован трассировщик лучей на CUDA с экспортом изображения в BMP.

Программа поддерживает два режима задания сцены:

1. Загрузка сцены из OBJ/MTL.
2. Случайная генерация сцены через CURAND, если файл сцены не передан.

Текущая реализация рендерит сферы и треугольники, считает прямое освещение с тенями и поддерживает вторичные лучи (отражение и преломление) с ограниченной глубиной.
За счет работы с треугольниками можно отрендерить практически лбой объект, сгенерировав треугольники. По такому принципу работает чтение .obj файлов с библиотекой материалов .mtl
В MTL поддерживаются основные материалы.

## Реализованный пайплайн

Порядок выполнения программы:

1. Построение сцены на хосте:
- из OBJ/MTL (геометрия + материалы + источники света), или
- процедурно через CURAND.

2. Копирование массивов сцены на GPU:
- материалы,
- сферы и их material id,
- треугольники и их material id,
- источники света.

3. Запуск CUDA-ядра:
- один поток вычисляет один пиксель.

4. Копирование буфера пикселей обратно на хост и сохранение BMP.

5. Вывод времени выполнения GPU-ядра.

## Математическая модель

### Генерация луча камеры

метод определения векторов $\mathbf{right}$ и $\mathbf{up}$ позаимоствован [отсюда](https://www.scratchapixel.com/lessons/mathematics-physics-for-computer-graphics/lookat-function/framing-lookat-function.html)
а генерация лучей частично взята [отсюда](https://www.scratchapixel.com/lessons/3d-basic-rendering/ray-tracing-generating-camera-rays/generating-camera-rays.html)
За все это в коде отвечает [камера](src/camera.h)

Для каждого пикселя $(x, y)$ изображения размера $(W, H)$:

$$
p_x = \left(2\frac{x + 0.5}{W} - 1\right) \cdot \text{aspect} \cdot \tan\left(\frac{\text{fov}}{2}\right)
$$

$$
p_y = \left(1 - 2\frac{y + 0.5}{H}\right) \cdot \tan\left(\frac{\text{fov}}{2}\right)
$$

$$
\mathbf{d} = \text{normalize}(\mathbf{forward} + p_x\mathbf{right} + p_y\mathbf{up})
$$

Первичный луч:

$$
\mathbf{r}(t) = \mathbf{o} + t\mathbf{d}
$$

### Пересечение со сферой

Для сферы с центром $\mathbf{c}$ и радиусом $R$:

$$
\|\mathbf{o}+t\mathbf{d}-\mathbf{c}\|^2 = R^2
$$

Получаем квадратное уравнение:

$$
at^2 + bt + c = 0
$$

где

$$
a = \mathbf{d}\cdot\mathbf{d}, \quad b = 2(\mathbf{o}-\mathbf{c})\cdot\mathbf{d}, \quad c = (\mathbf{o}-\mathbf{c})\cdot(\mathbf{o}-\mathbf{c}) - R^2
$$

Используется ближайший положительный корень.

### Пересечение с треугольником

Для треугольника используется алгоритм Меллера-Трумбора

### Нормали

Для треугольников завезена интерполяция нормалей по барицентрическим координатам, если такие лежат в obj файле. Если не лежат, то там будет обычная нормаль к плоскости. Это позволяет рендерить плавные картинки без артефактов треугольников.

### Модель освещения [Phong](https://en.wikipedia.org/wiki/Phong_reflection_model)

Локальный цвет начинается с фоновой и эмиссивной составляющих:

$$
\mathbf{C}_{local} = \mathbf{K_a} + \mathbf{K_e}
$$

где:
- $\mathbf{C}_{local}$ - итоговый локальный цвет (RGB-вектор),
- $\mathbf{K_a}$ - коэффициент фонового освещения ([ambient](src/material.h#L19) материала),
- $\mathbf{K_e}$ - цвет самосвечения  ([intensity](src/material.h#L42) материала).

Для каждого видимого точечного источника света:

1. Диффузная составляющая

$$
I_d = \max(0, \mathbf{n}\cdot\mathbf{l})
$$

$$
\mathbf{C}_{diff} = \mathbf{I}_{light} \odot \mathbf{K_d} \cdot I_d
$$

где:
- $\mathbf{n}$ - нормаль к поверхности в точке пересечения ,
- $\mathbf{l}$ - вектор от точки пересечения к источнику света,
- $\mathbf{I}_{light}$ - цвет ([интенсивность источника света](src/light.h#L7)),
- $\mathbf{K_d}$ - [коэффициент диффузного отражения материала](src/material.h#L27),
- $\odot$ - поэлементное умножение векторов.


2. Зеркальная составляющая

$$
\mathbf{r} = \text{reflect}(\mathbf{d}, \mathbf{n})
$$

$$
I_s = \max(0, \mathbf{l}\cdot\mathbf{r})^{\alpha}
$$

$$
\mathbf{C}_{spec} = \mathbf{I}_{light} \odot \mathbf{K_s} \cdot I_s
$$

где:
- $\text{reflect}(\mathbf{d}, \mathbf{n})$ - функция отражения: $\mathbf{r} = 2(\mathbf{d}\cdot\mathbf{n})\mathbf{n} - \mathbf{d}$,
- $\mathbf{d}$ - направление от источника к точке
- $\alpha$ - размытость блика ([в коде specular_exponent](src/material.h#L56)),
- $\mathbf{K_s}$ - [коэффициент отражения материала](src/material.h#L35).

Также дополнительно я добавил [albedo](src/material.h#L49), он используется как вектор весов для разных вкладов в итоговый цвет:

- $albedo[0]$ - вес прямого освещения (диффузная + зеркальная составляющие от источников света),
- $albedo[1]$ - вес отраженных лучей,
- $albedo[2]$ - вес преломленных лучей.

Поэтому для локального освещения в коде фактически используется:

$$
\mathbf{C}_{local} = \mathbf{K_a} + \mathbf{K_e} + albedo[0] \cdot (\mathbf{C}_{diff} + \mathbf{C}_{spec})
$$

А полный цвет точки можно записать в виде:

$$
\mathbf{C} \approx \mathbf{C}_{local} + albedo[1] \cdot \mathbf{C}_{refl} + albedo[2] \cdot \mathbf{C}_{refr}
$$

То есть albedo управляет балансом между прямым освещением, отражением и преломлением.

### Проверка тени

Для каждого источника строится луч из точки пересечения со сдвигом на epsilon.
Если расстояние до ближайшего пересечения меньше расстояния до источника, вклад этого света отбрасывается.

### Модель преломления

Направление преломленного луча вычисляется в векторной форме закона Снеллиуса:

$$
\mathbf{r}_{\perp} = \eta(\mathbf{d} + \cos\theta\,\mathbf{n})
$$

$$
\mathbf{r}_{\parallel} = -\sqrt{1 - \|\mathbf{r}_{\perp}\|^2}\,\mathbf{n}
$$

$$
\mathbf{d}_{refr} = \mathbf{r}_{\perp} + \mathbf{r}_{\parallel}
$$

Если подкоренное выражение отрицательно, возникает полное внутреннее отражение и преломленный луч не добавляется.

## Через что пришлось пройти и что сделано

### 1. Формат хранения и передачи сцены

Сцена на устройстве хранится в виде наборов буферов:

- непрерывный массив материалов,
- непрерывный массив сфер + массив их material id,
- непрерывный массив треугольников + массив их material id,
- непрерывный массив источников света.


### 2. Итеративный TraceRay

Рекурсивная трассировка на GPU заменена на стек фиксированного размера (у меня рекурсия падала)

### 3. Численная устойчивость

При запуске вторичных и теневых лучей применяется малый epsilon-сдвиг для уменьшения самопересечений (иногда были артефакты)

### 4. Генерация случайной сцены через CURAND

Генератор случайной сцены использует CURAND:

- создание псевдослучайного генератора,
- генерация массива равномерных double на device,
- копирование массива случайных значений на host,
- построение материалов и параметров сфер по диапазонам.

## Сборка и запуск

### Сборка

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target lab3_raytracer -j
```

### Примеры запуска

```bash
# Сцена из OBJ
./build/lab3-raytracer/lab3_raytracer scene.obj out.bmp 1280 720 5

# Случайная сцена через CURAND (без файла сцены)
./build/lab3-raytracer/lab3_raytracer out.bmp 1280 720 5

# Запуск со значениями по умолчанию
./build/lab3-raytracer/lab3_raytracer
```

Формат:

```text
lab3_raytracer [scene.obj] [output.bmp] [width] [height] [depth]
```
